# WORK_LOG.md

> Work log for SmartSAM. Compact history + active context so new chat sessions can pick up immediately.

## Rules
- Follow `AGENTS.md` as the strict rulebook.
- Keep this file as the project work log and update it before starting a task and after finishing a task.
- For every feature being prepared or about to be implemented, create a separate implementation plan under `Documents/ImplementationPlans/` and add its file reference here before coding starts.
- Never invent legacy helpers, stored procedures, SQL functions, table columns, or business rules. If VB references something missing, stop and ask.
- VB is source of truth for business logic; STContract governs code pattern/naming/structure.
- No fallback/heuristic/data-repair logic for legacy data unless explicitly approved.
- Only rebuild/restart Docker when user asks or clearly necessary.
- Code style is intentionally low-level (raw SQL + Dapper, inline PageModel logic, no service/DTO/repository abstraction) — PM decision for readability.
